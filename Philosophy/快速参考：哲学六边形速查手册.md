# 快速参考：哲学六边形速查手册

## 一、哲学六边形核心概念速查

### 1. 认识论 (Epistemology)
**核心问题**：我们如何知道我们知道什么？
**关键概念**：
- 地图 vs 疆域
- 社会建构
- 怀疑精神
- 认知开放性

**一句话总结**：保持怀疑，区分事实与虚构。

### 2. 科学哲学 (Philosophy of Science)
**核心问题**：科学如何认识世界？
**关键概念**：
- 模型思维
- 理论欠定性
- 多模型实在论
- 证伪主义

**一句话总结**：所有模型都是错的，但有些有用。

### 3. 实用主义 (Pragmatism)
**核心问题**：什么在实践中有用？
**关键概念**：
- 贝叶斯更新
- MVP思维
- 结果导向
- 快速迭代

**一句话总结**：不要争论真理，去测试它。

### 4. 现象学 (Phenomenology)
**核心问题**：我们如何体验世界？
**关键概念**：
- 意向性
- 生活世界
- 在场体验
- 悬置偏见

**一句话总结**：关注人的感受，而非物理参数。

### 5. 辩证法 (Dialectics)
**核心问题**：事物如何变化发展？
**关键概念**：
- 正题-反题-合题
- 否定之否定
- 矛盾统一
- 趋势预见

**一句话总结**：从矛盾中预见未来。

### 6. 伦理学/美学 (Ethics/Aesthetics)
**核心问题**：什么是善？什么是美？
**关键概念**：
- 功利主义 vs 义务论
- 技术伦理
- 形式与功能
- 品味与崇高

**一句话总结**：用善引导技术，用美打动人心。

## 二、问题诊断与哲学工具匹配

### 当你遇到这些问题时...

#### 问题类型1：行业惯例束缚
**症状**："一直都是这么做的"、"行业标准就是这样"
**推荐工具**：认识论 + 第一性原理
**具体行动**：
1. 质疑：这真的是唯一方式吗？
2. 拆解：基本事实是什么？
3. 重构：有没有更好的方式？

#### 问题类型2：技术瓶颈
**症状**：性能达到极限、成本无法降低
**推荐工具**：科学哲学 + TRIZ
**具体行动**：
1. 多模型：尝试不同技术路径
2. 矛盾分析：识别核心矛盾
3. 创新原理：应用TRIZ解决方案

#### 问题类型3：用户不接受
**症状**：功能完善但用户不用、满意度低
**推荐工具**：现象学 + 设计思维
**具体行动**：
1. 深度观察：用户真实使用场景
2. 同理心：理解用户感受
3. 原型测试：快速验证改进

#### 问题类型4：方向迷茫
**症状**：不知道下一步该做什么、趋势看不清
**推荐工具**：辩证法 + 趋势分析
**具体行动**：
1. 正反分析：当前趋势及其反面
2. 矛盾识别：主要矛盾是什么？
3. 合题预见：融合方向是什么？

#### 问题类型5：团队动力问题
**症状**：缺乏激情、价值观冲突、美感缺失
**推荐工具**：伦理学 + 美学 + 存在主义
**具体行动**：
1. 意义讨论：我们为什么做这个？
2. 价值共识：我们的伦理准则是什么？
3. 美感提升：如何让工作更美好？

#### 问题类型6：决策困难
**症状**：信息不足、风险高、选择困难
**推荐工具**：实用主义 + 贝叶斯
**具体行动**：
1. MVP测试：最小成本验证
2. 数据收集：获取反馈信息
3. 信念更新：调整决策模型

## 三、创新范式与哲学映射速查

### 四大创新范式

#### 1. 设计思维 → 现象学封装
**何时用**：解决人的问题、用户体验设计
**核心步骤**：同理心→定义→构思→原型→测试
**哲学内核**：现象学还原、生活世界理解

#### 2. 第一性原理 → 科学哲学封装
**何时用**：技术颠覆、成本重构、行业突破
**核心步骤**：解构→识别公理→重构→创新
**哲学内核**：还原论、模型思维

#### 3. 精益创业 → 实用主义封装
**何时用**：市场验证、商业模式探索、快速迭代
**核心步骤**：构建→衡量→学习
**哲学内核**：贝叶斯更新、证伪主义

#### 4. TRIZ → 辩证法封装
**何时用**：解决技术矛盾、工程创新
**核心工具**：40发明原理、矛盾矩阵
**哲学内核**：矛盾统一、正反合

## 四、实践检查清单

### 每日检查（5分钟）

- [ ] 今天我质疑了什么假设？（认识论）
- [ ] 我用了什么模型思考问题？（科学哲学）
- [ ] 我根据什么新信息更新了信念？（实用主义）
- [ ] 我观察到了什么用户感受？（现象学）
- [ ] 我预见了什么趋势变化？（辩证法）
- [ ] 我的决策考虑了哪些价值？（伦理学/美学）

### 每周复盘（30分钟）

**本周哲学应用评估**：
1. **最强维度**：哪个哲学维度用得最好？
2. **最弱维度**：哪个维度需要加强？
3. **关键洞察**：本周最重要的哲学洞察是什么？
4. **改进计划**：下周如何应用得更好？

### 月度深度反思（2小时）

**本月哲学成长**：
1. **知识积累**：学习了哪些新哲学概念？
2. **技能提升**：哪个哲学工具用得更加熟练？
3. **案例研究**：分析了哪些创新者的哲学应用？
4. **实践成果**：哲学思维带来了什么实际成果？

## 五、常见误区与纠正

### 误区1：过度哲学化
**表现**：每个决策都要哲学分析，行动迟缓
**纠正**：建立"战士模式"，90%时间专注执行

### 误区2：哲学炫耀
**表现**：用哲学术语装点门面，实际脱离实践
**纠正**：黑箱封装，只输出简单指令

### 误区3：维度偏食
**表现**：只擅长1-2个维度，忽视其他
**纠正**：定期自我评估，针对性加强薄弱维度

### 误区4：时机错配
**表现**：在错误阶段应用错误哲学
**纠正**：根据项目生命周期动态调整权重

### 误区5：知行分离
**表现**：哲学说得头头是道，行动完全另一套
**纠正**：从最小实践开始，逐步整合

## 六、紧急情况快速响应

### 危机类型1：重大失败
**哲学响应**：实用主义 + 存在主义
**具体行动**：
1. 接受现实（存在主义）
2. 分析原因（科学哲学）
3. 更新信念（贝叶斯）
4. 重新出发（实用主义）

### 危机类型2：团队冲突
**哲学响应**：伦理学 + 现象学
**具体行动**：
1. 理解各方立场（现象学）
2. 寻找共同价值（伦理学）
3. 建立对话框架（辩证法）
4. 达成新共识（合题）

### 危机类型3：方向迷失
**哲学响应**：辩证法 + 认识论
**具体行动**：
1. 质疑当前方向（认识论）
2. 分析趋势矛盾（辩证法）
3. 探索新可能性（多模型）
4. 选择新路径（实用主义）

### 危机类型4：道德困境
**哲学响应**：伦理学 + 辩证法
**具体行动**：
1. 识别利益冲突（伦理学）
2. 分析长期影响（辩证法）
3. 权衡不同价值（功利vs义务）
4. 做出负责任选择（价值论）
